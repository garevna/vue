Vue Router ходит по маршрутам...
На самом деле не совсем так
Роутер монтирует нужный компонент в нужный момент времени в нужном месте
Нужное место указывается с помощью плейсхолдера

<router-view></router-view>
Компоненты, которые будут смонтированы в "нужном месте", нужно привязать к маршрутам

{
    path: "/articles",
    component: ArticlesStore
}

Из таких объектов составляется массив

routes: [
     {
       path: '/library',
       component: ...
     },
     ...
}

('/' означает маршрут приложения по умолчанию)
и этот массив передается роутеру. Можно это сделать сразу при создании роутера:

const router = new VueRouter ( {
    routes: routes
})

Роутер нужно привязать к экземпляру Vue:

new Vue ({
  el: '#app',
  router,
  components: { ... }
})

Машинка роутера готова
Теперь осталось только связать всю эту машинку с UI
Для этого вставляем теги:

<router-link to = "/library"></router-link>
в html-код или в темплейт компонента (в общем, туда, где должна появиться ссылка на маршрут
После рендеринга этот тег будет заменен на обычную гиперссылку
Маршруты могут быть вложенными:
routes: [
     {
       path: '/...',
       name: '...',
       component: ...
     },
     {
       path: '/firstroute/:name',
       name: 'FirstRoute',
       component: FirstRoute,
       children: [
         {
           path: 'child',
           component: FirstRouteChild
         }
       ]
     }
   ]
